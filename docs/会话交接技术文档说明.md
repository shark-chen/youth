# 会话交接技术文档说明

> **文档性质**：面向希望理解或复现该模式的同学，说明设计思路与实现要点。  
> **不是**项目日常开发必读清单；日常协作仍以仓库内 `AGENTS.md` 与 `.cursor/rules/` 为准。

---

## 1. 要解决什么问题

在 AI 辅助编程（如 Cursor Agent）中，**单次对话有上下文上限**，且**新会话无法自动继承**上一轮的细节。常见痛点包括：

- 改动了哪些文件、做到哪一步，下次容易忘；
- 未跑测试、未合并分支、已知风险，容易在续作时重复踩坑；
- 多人或隔日续作时，缺少一段「可粘贴的冷启动说明」。

**会话交接（Session Handoff）** 指：在一段协作告一段落时，由 Agent **按固定结构输出一段摘要**，用户可粘贴到**下一次会话的首条消息**，或保存为本地文件，从而降低上下文断裂成本。

---

## 2. 设计目标

| 目标 | 说明 |
|------|------|
| **可触发** | 用户用自然语言（如「交接」「收尾」）即可触发，不必记复杂指令。 |
| **结构化** | 摘要包含目标、改动、待办、下一步等固定维度，减少遗漏。 |
| **可落盘（可选）** | 需要时**仅**写入本机已 ignore 的路径（如 `.cursor/session-handoff.md`），不写入其他可跟踪 `.md`（除非用户明确指定）。 |
| **不污染 Git** | 交接常含任务细节，默认 **不提交** 到远程，避免误泄露。 |
| **与编辑器集成** | 利用 Cursor 的 **Project Rules** 与 **Commands**，无需改业务代码。 |

### 2.1 发现即记录（agent-backlog）

为缓解 **Agentic Coding 中「新发现被上下文挤掉」**：在规则中要求将**超出当前任务**的观察**追加**到本机 **`.cursor/agent-backlog.md`**（gitignore），收工时在交接中增加 **「观察到但未处理」** 并与 backlog 对照。详见本仓库 **`.cursor/agent-backlog.template.md`** 与规则中的「发现即记录」小节；**独立说明**见 **`docs/发现即记录技术文档说明.md`**。

## 3. 整体架构（四件套）

实现上通常组合使用：

```text
┌─────────────────────────────────────────────────────────┐
│  Project Rule（.cursor/rules/*.mdc）                     │
│  → 约束 Agent：何时输出、必须包含哪些小节、篇幅与语言      │
└─────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐   ┌─────────────────┐   ┌──────────────────┐
│ Markdown 模板  │   │ .gitignore      │   │ Cursor Command   │
│ （可提交仓库）   │   │ 忽略本地交接文件 │   │ 一键唤起说明/步骤 │
└───────────────┘   └─────────────────┘   └──────────────────┘
```

**职责简述：**

1. **规则（Rule）**  
   - Cursor 会把 `alwaysApply: true` 等规则注入 Agent 上下文，使「收尾时写交接」成为**稳定行为**，而不是偶发提醒。

2. **模板（Template）**  
   - 提交到仓库的 **空骨架**（标题 + 占位符），统一章节名，方便人类与 Agent 对齐「什么叫一份完整交接」。

3. **本地交接文件 + gitignore**  
   - 实际填写内容放在例如 `.cursor/session-handoff.md`。  
   - 将该路径加入 `.gitignore`，避免误提交敏感或临时上下文。

4. **命令（Command）**  
   - 在 Cursor 命令面板中通过名称（如 `session-handoff`）触发，文档里写清步骤：先归纳对话与文件，再输出 Markdown，**仅在用户明确要求时再写入**本地文件，避免静默覆盖。

---

## 4. Cursor 侧机制（概念层）

- **Project Rules**（`.cursor/rules/*.mdc`）  
  项目级指令，影响本仓库内的 Agent 行为。适合放「必须遵守」的交接约定。  
  具体 YAML 头字段以当前 Cursor 文档为准（如 `description`、`alwaysApply`）。

- **Commands**（`.cursor/commands/*.md`）  
  可复用的提示词/步骤说明，降低用户每次手打一长段说明的成本。  
  本技能中的命令本质是：**指向规则与模板，并强调落盘条件**。

- **无「关窗自动执行」**  
  Cursor 一般不会在关闭窗口时自动跑 Agent，因此 **「交接」仍需用户主动触发**（话术或命令），这是设计上的现实约束，规则里应写清楚触发条件。

---

## 5. 规则里建议写清的内容

- **触发条件**：例如用户说「结束会话 / 交接 / 会话摘要 / 收尾」，或任务已交付且用户表示收工。  
- **输出格式**：简体中文、Markdown、必备小节列表（日期、目标与结论、涉及路径、未完成/风险、**任务计划进度**（若存在 `task-plan.md`）、下次第一步、**观察到但未处理**）。  
- **篇幅**：默认 200～400 字量级，避免过长占满上下文；需要细节时由用户要求展开。  
- **落盘策略**：仅在用户同意或明确要求时写入 `.cursor/session-handoff.md`（仅本机、已 ignore）；**发现类**条目平时追加到 `.cursor/agent-backlog.md`。  
- **发现即记录**：见 §2.1；落盘策略与「仅本机 md」约定见仓库规则。

---

## 6. 迁移到其他项目的步骤

1. 在目标仓库新增 `.cursor/rules/` 下一条规则（或合并进现有规则），写入上一节的触发条件与章节要求。  
2. 新增可提交的 `session-handoff.template.md`（或放在 `docs/` 下命名清晰）。  
3. 将 `.cursor/session-handoff.md`（或你自定义的路径）加入 `.gitignore`。  
4. 可选：在 `.cursor/commands/` 增加 `session-handoff` 命令说明。  
5. 在团队内部约定：**新会话续作时**，优先把上次的交接粘贴到首条消息，或 `@` 本地交接文件（若 Cursor 版本支持该工作流）。

---

## 7. 使用与注意事项

- **隐私**：交接可能包含分支名、接口名、内部路径；gitignore 本地文件 + 不随截图外发敏感段落。  
- **准确性**：摘要依赖对话与 Agent 对改动的理解，**关键结论仍应以 Git diff / CI 为准**。  
- **与 Memory 的关系**：若使用 Cursor Memory，交接可与 Memory 互补：Memory 偏长期偏好，交接偏**当前任务切片**。

---

## 8. 局限与改进方向

| 局限 | 可能改进 |
|------|----------|
| 需手动触发 | 在规则中列出明确口令；团队统一用命令名。 |
| 长项目多并行任务 | 交接里写清分支名、Issue 编号；或按任务分子目录命名文件（注意仍建议 ignore）。 |
| 与 CI 状态不同步 | 交接中显式写「未跑 analyze / 未跑测试」等。 |
